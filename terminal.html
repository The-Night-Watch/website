Project Terminal (2020)

Open source portable music player

Project
Terminal

Frustrated by the lack of options for a simple, portable audio player, I decided to design my own, open source, and based on the Raspberry Pi Zero W. The project is still a work in progress, and I maintain a git repository of the project files not quite user friendly for the moment but I'll work on proper documentation soon.

It is my first attempt at designing a physical product, and I am learning a lot in the process. It is an interesting exercise, with its own problem space, different from that of graphic design. One of the fascinating aspects of it is the duality between internal hardware and its external interface. Each choice made in one affecting the other. This requires a a flexible iterative workflow and a lot of patience, making the task challenging but rewarding. After some trial and error, it became clear that a purely bottom up approach would be too limiting. Simply making a list of the ideal internal components and design around that made for quite a boring and unattractive design.

I quickly decided to make cardboard models for each design iteration. It might sound obvious, but it was a major shift in my process. Suddenly, the buttons naturally found their place on the front plate, I rethought the way I had planned the volume controls, and I got, quite literally, a better feel for how the device should respond to interaction.

This top down approach helped a lot for the buttons layout, but in a similar way, hardware can also influence the productâ€™s external design in a big way. Browsing electronic components catalogs offers endless ways of solving a particular problem and avoid others entirely.

When I discovered the existence of affordable round OLED displays, it opened up more design possibilities. I was able to make the front plate of the device symmetrical. Not only does it look cool, but it makes it possible to use the device in any orientation, simply by rotating the UI in software, which was a great step towards better accesibility.

The same pattern happened again with the volume control interface. I considered many different options, but after a lot of research on online electronics shops, I ended up going with a potentiometer switch, similar to those found on physical synthetisers. It was a great way, if a bit unconventional on this type of device, to allow for a precise and organic feel when adjusting volume. Apple came to the same conclusion for their headphones, which they unveiled a few months later.

The last step before the production of a small number of test units is figuring out which option to go with for the casing. I will keep this page and the git repo updated with future progress.
